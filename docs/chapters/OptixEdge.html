Deploy your first container on your OptixEdge

Follow this tutorial to deploy your first MQTT broker on OptixEdge using Docker. The procedure is based on the OptixEdge & Docker Containers presentation (page 42 onward).

Before loading any container: enable Docker Engine from System Manager. Open https://optix_edge_ip_address, sign in, go to Docker, enable Docker Engine under Services, then reboot OptixEdge.

Load from USB key

  1. Use a USB key formatted as FAT32, exFAT, or ext4.
  2. Pull the container image for linux/arm64.
  3. Export the image as a .tar file.
  4. Create a docker-compose.yaml that defines startup options.
  5. Copy both files to the USB key root directory.
  6. Plug the USB key into OptixEdge.
  7. In System Manager, open Docker and click Import docker image via USB in the Containers area.
On your machine 
docker pull --platform linux/arm64 eclipse-mosquitto:2
docker save -o mosquitto.tar eclipse-mosquitto:2
docker-compose.yaml 
services:
  mosquitto:
    image: eclipse-mosquitto:2
    container_name: mosquitto
    ports:
      - "1883:1883"
      - "9001:9001"
    restart: unless-stopped

Deploy with Portainer

  1. In System Manager, go to Docker and enable Portainer CE under Services.
  2. Reboot OptixEdge.
  3. Open https://optix_edge_ip_address:9443 and trust the self-signed certificate.
  4. Set the admin password at first login.
  5. Select the local environment, open Stacks, then click Add stack.
  6. Paste the docker-compose.yaml content from Path 1 and deploy the stack.

Deploy with Docker CLI remote management

  1. In System Manager, under Docker > Remote Management, choose the connection type and allowed network interfaces.
  2. On your local machine, set DOCKER_HOST to target OptixEdge.
  3. Run standard Docker CLI commands against OptixEdge, then unset the variable to restore local behavior.
Linux shell example 
export DOCKER_HOST=tcp://optix_edge_ip:2375
docker ps
unset DOCKER_HOST
Windows PowerShell example 
$env:DOCKER_HOST = "tcp://optix_edge_ip:2375"
docker ps
Remove-Item Env:DOCKER_HOST

      - "3001:3001"
    volumes:
      - kuma_data:/app/data
    restart: unless-stopped

  kuma_data:

MySQL + phpMyAdmin: deploy a database plus a web UI to practice persistence and service linking.

Portainer stack 
version: '3.8'

services:
  mysql:
    image: mysql:8
    container_name: mysql-retention
    environment:
      MYSQL_ROOT_PASSWORD: Password01 # Change to a strong password
      MYSQL_DATABASE: example_db
      MYSQL_USER: user
      MYSQL_PASSWORD: Password01 # Change to a strong password
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    restart: unless-stopped

  phpmyadmin:
    image: phpmyadmin:latest
    container_name: phpmyadmin
    depends_on:
      - mysql
    environment:
      PMA_HOST: mysql
      PMA_PORT: 3306
      MYSQL_ROOT_PASSWORD: Password01 # Keep in sync with MYSQL_ROOT_PASSWORD
    ports:
      - "8090:80"
    restart: unless-stopped

volumes:
  mysql_data:

Gemma 3 (1B): run Ollama with the smallest Gemma 3 model for lightweight local AI experiments.

Portainer stack 
version: '3.8'

services:
  ollama:
    image: docker.io/ollama/ollama:latest
    container_name: ollama-gemma3-1b
    ports:
      - "11434:11434"
    volumes:
      - ollama_data:/root/.ollama
    command: >
      /bin/sh -c "ollama serve &
      sleep 5;
      ollama list | grep -q 'gemma3:1b' || ollama pull gemma3:1b;
      wait"
    restart: unless-stopped

volumes:
  ollama_data:

Verify the MQTT broker

  1. Confirm that the container is running and port 1883 is published.
  2. On Windows, install the Mosquitto client tools from https://mosquitto.org/download/ and open two PowerShell terminals in the Mosquitto installation directory.
  3. From a client machine with mosquitto-clients, subscribe to a test topic.
  4. Publish one message to the same topic.
  5. Verify that the subscriber receives the message.
Windows PowerShell (subscriber) 
.\mosquitto_sub.exe -h optix_edge_ip_address -p 1883 -t demo/edge/test
Windows PowerShell (publisher) 
.\mosquitto_pub.exe -h optix_edge_ip_address -p 1883 -t demo/edge/test -m "optixedge mqtt ok"
Subscriber terminal 
mosquitto_sub -h optix_edge_ip_address -p 1883 -t demo/edge/test
Publisher terminal 
mosquitto_pub -h optix_edge_ip_address -p 1883 -t demo/edge/test -m "optixedge mqtt ok"
Expected output on subscriber 
optixedge mqtt ok

Docker logging tip

Set log rotation in each service to avoid filling local storage.

Minimal example 
logging:
  driver: "json-file"
  options:
    max-size: "10m"
    max-file: "3"
