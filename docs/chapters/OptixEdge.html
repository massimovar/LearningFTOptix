Deploy your first container on your OptixEdge

Follow this tutorial to deploy your first MQTT broker on OptixEdge using Docker. The procedure is based on the OptixEdge & Docker Containers presentation (page 42 onward).

Before loading any container: enable Docker Engine from System Manager. Open https://optix_edge_ip_address, sign in, go to Docker, enable Docker Engine under Services, then reboot OptixEdge.

Load from USB key

  1. Use a USB key formatted as FAT32, exFAT, or ext4.
  2. Pull the container image for linux/arm64.
  3. Export the image as a .tar file.
  4. Create a docker-compose.yaml that defines startup options.
  5. Copy both files to the USB key root directory.
  6. Plug the USB key into OptixEdge.
  7. In System Manager, open Docker and click Import docker image via USB in the Containers area.
On your machine 
docker pull --platform linux/arm64 eclipse-mosquitto:2
docker save -o mosquitto.tar eclipse-mosquitto:2
docker-compose.yaml 
services:
  mosquitto:
    image: eclipse-mosquitto:2
    container_name: mosquitto
    ports:
      - "1883:1883"
      - "9001:9001"
    restart: unless-stopped

Deploy with Portainer

  1. In System Manager, go to Docker and enable Portainer CE under Services.
  2. Reboot OptixEdge.
  3. Open https://optix_edge_ip_address:9443 and trust the self-signed certificate.
  4. Set the admin password at first login.
  5. Select the local environment, open Stacks, then click Add stack.
  6. Paste the same docker-compose.yaml content shown above and deploy the stack.

Deploy with Docker CLI remote management

  1. In System Manager, under Docker > Remote Management, choose the connection type and allowed network interfaces.
  2. On your local machine, set DOCKER_HOST to target OptixEdge.
  3. Run standard Docker CLI commands against OptixEdge, then unset the variable to restore local behavior.
Linux shell example 
export DOCKER_HOST=tcp://optix_edge_ip:2375
docker ps
unset DOCKER_HOST
Windows PowerShell example 
$env:DOCKER_HOST = "tcp://optix_edge_ip:2375"
docker ps
Remove-Item Env:DOCKER_HOST

Verify the MQTT broker

  1. Confirm that the container is running and port 1883 is published.
  2. From a client machine with mosquitto-clients, subscribe to a test topic.
  3. Publish one message to the same topic.
  4. Verify that the subscriber receives the message.
Subscriber terminal 
mosquitto_sub -h optix_edge_ip_address -p 1883 -t demo/edge/test
Publisher terminal 
mosquitto_pub -h optix_edge_ip_address -p 1883 -t demo/edge/test -m "optixedge mqtt ok"
Expected output on subscriber 
optixedge mqtt ok
